# Spec: Daily Planner + AI Check-ins + Recurring Tasks + Subtask Trees

## Objective

Build four interconnected features that transform the AI Assistant Console from a passive inbox/board into a proactive, ADHD-friendly daily planning system — closing the biggest feature gaps vs. saner.ai.

**Users:** Single user (sam) — personal productivity tool.

**Success looks like:** Opening the app each morning shows an AI-generated briefing of what's on your plate, with recurring tasks already created, stale items flagged, and overdue items surfaced. Push notifications arrive twice a day even when the app is closed. Tasks can have unlimited subtask nesting for breaking down complex work.

---

- **Framework:** Next.js 14 (App Router) — existing
- **Database:** Supabase (PostgreSQL) — existing

## Feature 1: "My Day" Page

### Route: `/app/my-day`

### Server Component (`page.tsx`)
- Fetch all items where `status = 'active'` and lane = Today (priority_score >= 0.85)
- Fetch items with `metadata.recurrence` where `next_due <= today` (should already be generated by cron, but fallback generate here too)
- Fetch overdue items: active items in Today with `created_at` or `updated_at` before today
- Fetch stale items: active items in Next/Backlog with `updated_at` older than 5 days
- Pass all to client component

### Client Component (`my-day-client.tsx`)

**Layout (top to bottom):**

1. **AI Briefing Card** (collapsible, open by default on first visit of the day)
   - Generated once per day, cached in `localStorage` keyed by date
   - Content:
     - Task count: "You have **6 tasks** today, **2 high priority**"
     - Top focus: "Your #1 focus: **Ship the API endpoint**" (highest priority_score item)
     - Overdue rollover: "⚠️ 2 items rolled over from yesterday" (with item titles, clickable)
     - Stale nudges: "💡 'Write docs' has been in Next for 5 days — move to Today?" (actionable button)
   - Briefing text generated by OARS (short, conversational, ADHD-friendly tone)
   - Falls back to structured template if OARS is unavailable

2. **Today's Task List**
   - All Today-lane active items, ordered by priority_score descending
   - Each item shows:
     - Checkbox (complete action)
     - Title (click to expand detail inline)
     - Recurrence badge: 🔄 Daily / 🔄 Weekly (if recurring)
     - Subtask progress: "2/4 subtasks done" (if has children)
     - Expandable subtask tree (see Feature 4)
   - Drag-to-reorder within the list (updates priority_score)

3. **Progress Bar**
   - "3 of 6 done today" — visual horizontal fill bar
   - Uses `--accent` color, animates on completion
   - Shows percentage

4. **Quick Add** (pinned at bottom)
   - Inline text input, Enter to capture
   - Items go directly to Today lane (priority_score = 0.90)
   - AI classifies type in background (same as inbox composer)

### Navigation
- New sidebar entry: **"My Day"** — positioned at the very top of the sidebar, above current lane buttons
- Uses a ☀️ icon or similar
- Active state: colored left border like lane buttons (use `--accent`)
- Clicking navigates to `/app/my-day` (full page transition, not filter change)
- The existing Board remains at `/app` as the full kanban view

## Feature 2: Recurring Tasks

### Data Model

Recurrence stored in `items.metadata` JSON (no schema migration needed):

```typescript
interface RecurrenceConfig {
  frequency: "daily" | "weekly";
  days?: number[];          // For weekly: 1=Mon, 2=Tue, ..., 7=Sun. Empty/absent for daily.
  next_due: string;         // ISO date: "2026-04-24"
  template_id?: string;     // On generated instances, points back to the template item
  is_template?: boolean;    // true on the original recurring item
}

// Stored as:
// item.metadata.recurrence = { frequency: "daily", next_due: "2026-04-24", is_template: true }
```

### UI: Setting Recurrence

In the **detail panel** (board-client.tsx) and in **My Day inline edit**:
- New section below tags: **"Repeat"**
- Toggle: Off / Daily / Weekly
- If Weekly: day-of-week picker (Mon-Sun checkboxes, multiple select)
- Setting recurrence marks the item as `is_template: true` and sets `next_due` to tomorrow (daily) or next matching day (weekly)

### Generation Logic

**Cron API Route:** `POST /api/cron/generate-recurring`

- Secured with a `CRON_SECRET` env var (request must include `Authorization: Bearer $CRON_SECRET`)
- Queries: all items where `metadata->>'recurrence' IS NOT NULL` and `metadata->'recurrence'->>'is_template' = 'true'` and `metadata->'recurrence'->>'next_due' <= today`
- For each template:
  - Check if an instance already exists for today (prevent duplicates): look for active items with `metadata->'recurrence'->>'template_id' = template.id` and `created_at::date = today`
  - If no instance exists: INSERT new item copying title, type, content, tags from template
    - `status: 'active'`
    - `priority_score: 0.90` (Today lane)
    - `metadata.recurrence.template_id = template.id` (link back)
    - `metadata.recurrence.is_template = false`
  - Update template's `next_due`:
    - Daily: tomorrow
    - Weekly: next matching day from `days[]` array

**Trigger:**
- Primary: Coolify cron job hitting the endpoint daily at 00:05 UTC
- Fallback: When My Day page loads, if no recurring instances exist for today, run generation inline (same logic)

### Server Action: `setRecurrence(itemId, config | null)`
- Updates `metadata.recurrence` on the item
- If `config` is null, removes recurrence (clears template flag)
- Revalidates `/app` and `/app/my-day`

## Feature 3: Proactive AI Check-ins + Push Notifications

### Push Subscription Infrastructure

**New table: `push_subscriptions`**

```sql
CREATE TABLE push_subscriptions (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id TEXT NOT NULL,
  endpoint TEXT NOT NULL UNIQUE,
  p256dh TEXT NOT NULL,
  auth TEXT NOT NULL,
  created_at TIMESTAMPTZ DEFAULT now()
);
CREATE INDEX idx_push_sub_user ON push_subscriptions(user_id);
```

**VAPID Keys:**
- Generated once, stored as env vars: `VAPID_PUBLIC_KEY`, `VAPID_PRIVATE_KEY`, `VAPID_SUBJECT` (mailto: email)
- Public key exposed via a server action or inline in the page for client subscription

**API Routes:**

`POST /api/push/subscribe`
- Body: `{ endpoint, keys: { p256dh, auth } }`
- Upserts into `push_subscriptions` for current user

`POST /api/push/unsubscribe`
- Body: `{ endpoint }`
- Deletes from `push_subscriptions`

**Service Worker Enhancement (`sw.js`):**

```javascript
// New event handlers added to existing sw.js:
self.addEventListener('push', (event) => {
  const data = event.data?.json() ?? {};
  event.waitUntil(
    self.registration.showNotification(data.title || 'AI Assistant', {
      body: data.body,
      icon: '/icons/icon-192.png',
      badge: '/icons/icon-192.png',
      tag: data.tag || 'checkin',
      data: { url: data.url || '/app/my-day' }
    })
  );
});

self.addEventListener('notificationclick', (event) => {
  event.notification.close();
  event.waitUntil(
    clients.openWindow(event.notification.data.url)
  );
});
```

**Client-side subscription (in board-client.tsx or a settings component):**
- New settings toggle: "Enable push notifications"
- On enable: `Notification.requestPermission()` → register with service worker → POST to `/api/push/subscribe`
- On disable: unregister → POST to `/api/push/unsubscribe`
- Store preference in `localStorage` for UI toggle state

### Check-in Logic

**Cron API Route:** `POST /api/cron/send-checkins`

- Secured with `CRON_SECRET`
- Accepts query param `?type=morning` or `?type=evening`
- **Morning check-in (09:00 UTC — hardcoded for v1, configurable timezone later):**
  - Fetch today's active items (Today lane)
  - Fetch overdue (items in Today from previous days)
  - Fetch stale (items in Next/Backlog older than 5 days, limit 3)
  - Generate briefing via OARS:
    - System prompt: "You are a friendly ADHD-aware productivity assistant. Generate a brief morning check-in (2-3 sentences max). Be encouraging, specific about the user's tasks, and mention any overdue items gently."
    - User content: JSON of task summaries
  - Send push notification with generated text
  - Notification title: "☀️ Good morning!"
  - Click opens `/app/my-day`

- **Evening check-in (18:00 UTC — hardcoded for v1):**
  - Fetch today's completed items count vs total
  - Fetch remaining active items
  - Generate recap via OARS:
    - "Generate a brief end-of-day recap (2-3 sentences). Celebrate what was done. If items remain, suggest rolling them to tomorrow. Be warm and non-judgmental."
  - Send push notification
  - Notification title: "🌙 Day's wrap-up"
  - Click opens `/app/my-day`

**Cron schedule (Coolify or Hetzner cron):**
```
5 9 * * * curl -X POST -H "Authorization: Bearer $CRON_SECRET" https://app.example.com/api/cron/send-checkins?type=morning
0 18 * * * curl -X POST -H "Authorization: Bearer $CRON_SECRET" https://app.example.com/api/cron/send-checkins?type=evening
```

## Feature 4: Subtask Trees (Unlimited Nesting)

### Data Model

Subtasks are regular items with `metadata.parent_item_id` set (already exists in types.ts).

```typescript
// In ItemMetadata (lib/items/types.ts):
interface ItemMetadata {
  parent_item_id?: string;    // Already exists
  // ... other fields
}
```

No schema migration needed — parent_item_id is already used by `createSubtaskFromSuggestion`.

### Fetching the Tree

**Helper function:** `buildSubtaskTree(items: Item[]): TreeNode[]`
- Input: flat array of items (all items for the user, or a filtered set)
- Output: nested tree structure
- Algorithm: Group by parent_item_id, recursively build children arrays
- Items with no parent_item_id are roots

```typescript
interface TreeNode {
  item: Item;
  children: TreeNode[];
  depth: number;
}
```

### UI: Subtask Tree Component

**`SubtaskTree` component** (new file: `app/app/subtask-tree.tsx`):
- Renders a recursive tree of items
- Each node:
  - Left indent: `depth * 20px` padding
  - Collapse/expand toggle (▶/▼) if has children
  - Checkbox to complete
  - Title (editable inline on click)
  - "+ Add subtask" button (appears on hover)
- Completing a leaf updates its status
- When all children of a parent are completed: show subtle prompt "All subtasks done — complete parent?"

**Used in:**
1. **My Day page** — under each Today item, expandable
2. **Detail panel** (board-client.tsx) — below item content, above tags

### Server Actions

**`createSubtask(parentId: string, title: string)`**
- Creates new item with `metadata.parent_item_id = parentId`
- Inherits parent's lane (priority_score) and type
- `status: 'active'`, `needs_review: false`

**`reorderSubtasks(parentId: string, orderedIds: string[])`**
- Updates a `metadata.subtask_order` field on the parent to preserve manual ordering

## Boundaries

### Always:
- Run `npm test` before any commit
- Run `npx tsc --noEmit` to check types
- Use existing design tokens from DESIGN.md
- Store all new data in `metadata` JSON to avoid schema migrations where possible
- Secure cron endpoints with `CRON_SECRET`
- Revalidate paths after mutations

### Ask first:
- Adding new Supabase tables (push_subscriptions is pre-approved)
- Changing the items table schema directly
- Modifying the service worker caching strategy

### Never:
- Commit VAPID private keys or CRON_SECRET to git
- Send push notifications without user opt-in
- Auto-complete parent items when children are done (only suggest)
- Break existing board functionality

## Assumptions

1. Single-user app (hardcoded "sam") — no multi-user push subscription routing needed
2. User's timezone is known or can be inferred (default UTC, configurable later)
3. OARS API is available for briefing generation (graceful fallback to template if not)

5. The `metadata` JSONB column on items is flexible enough for recurrence + subtask data (confirmed: it's already used this way)
6. `web-push` npm package works in the Next.js server environment
